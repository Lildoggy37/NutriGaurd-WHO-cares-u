# NutriGuard RAG 检索引擎 — 离线评测报告

**评测日期**: 2025-05-30
**评测工具**: `rag_benchmark.py`

## 5. 语义缓存效果

```
第一次 "糖尿病可以吃什么？"  → Cache Miss → RAG 检索 (~80s)
第二次 "糖尿病可以吃什么？"  → Cache Hit  → Redis 返回 (<1ms)
第三次 "高血糖要注意啥？"    → Cache Hit  → 语义匹配命中 (<1ms)
```

缓存基于 BGE 1024 维向量 + 余弦相似度 (threshold=0.85)，同义改写可命中。

## 7. 关键发现与改进建议

### 优势

- **双路召回 + Reranker** 架构在标准查询和语义等价测试上表现优异，Recall@3 达 90%
- 所有命中查询均为 Top-1 位置，排序精度高
- 语义缓存可将重复/等价查询加速 **80,000x**

### 待改进

| 问题 | 影响 | 建议 |
|------|------|------|
| 语料覆盖不足（无减肥热量、反式脂肪酸） | Recall 90% 而非 100% | 扩充语料到 500+ 行，覆盖更多营养学主题 |
| 单线程推理延迟过高 | 首次查询 ~80s | GPU 推理或 ONNX 量化模型可降至 2-5s |
| 无多模型对比基线 | 无法量化 BGE+Reranker 的相对增益 | 添加 BM25-only、Dense-only 消融实验 |
